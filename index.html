Шпаргалка

Перелік основних команд:

  • git config --global user.name - встановлює глобально ім'я
  • git config --global user.email - встановлює глобально пошту
  • git init - ніціює пустий репозиторій в поточній директорії
  • git status - показує стан файлів в робочій директорії і індексі
  • git add - додає вміст робочої директорії в індекс (staging area) для подальших коммітов
  • git commit - Команда git commit бере всі дані, додані в індекс за допомогою git add, і зберігає їх зліпок у внутрішній базі даних, а потім зрушує покажчик поточної гілки на цей зліпок.
  • git log - виводить список коммітов створених в даному репозиторії в зворотному хронологічному порядку
  • git checkout - використовується для копіювання файлів з історії або сцени в робочу директорію
  • git branch -створює,перейменовує,видаляє, гілки а також перераховує їх
  • git push - використовується для встановлення зв'язку з віддаленим репозиторієм, обчислення локальних змін відсутніх в ньому, і власне їх передачі в вищезгаданий репозиторій.
  • git pull - спочатку забирає зміни із зазначеного віддаленого сховища, а потім намагається злити їх з поточної гілкою.
  • git merge - використовується для злиття однієї або декількох гілок в поточну
  • git clone - клонує віддалений репозиторій

.gitignore - служить для вказівки в ньому файлів і папок, які необхідно приховати від системи контролю версій git.

Команди:

  • git revert - створює "антікомміт" для зазначеного коммітів, таким чином скасовуючи зміни, внесені в ньому
  • git reset - використовується для скасування змін. Вона змінює покажчик HEAD і, опціонально, стан індексу. Також ця команда може змінити файли в робочій директорії при використанні параметра --hard.
  • git rebase - один з двох способів об'єднати зміни, зроблені в одній гілці, з іншого гілкою.
  • git cherry-pick - використовується для перенесення окремих коммітов з одного місця сховища в інше, зазвичай між гілками розробки та обслуговування
  • git stash - використовується для тимчасового збереження всіх незакомміченних змін для очищення робочої директорії без необхідності коммітов незавершену роботу в нову гілку.
